# Dryvre 에이전트 작업 규칙

## 문서의 권위와 구현 범위

- 현재 구현과 제품 결정은 `docs/README.md`가 연결하는 활성 문서를 따른다.
- `docs/not-implemented/`는 **현재 구현하지 않는 backlog**다.
- `docs/not-implemented/`의 문서는 설계 검토, 용어 확인, 향후 의사결정의 참고자료로 읽을 수 있다.
- backlog에 구체적인 타입, 함수, 상수, SQL 또는 구현 순서가 있어도 구현 승인으로 해석하지 않는다.
- 사용자가 해당 항목을 명시적으로 현재 범위로 승격하기 전에는 backlog를 근거로 코드, 스키마, 마이그레이션, API 또는 활성 스펙을 변경하지 않는다.
- backlog 항목이 현재 구현과 충돌하면 현재 구현과 활성 문서가 우선한다. 충돌 자체는 임의로 해소하지 말고 필요하면 사용자에게 알린다.

## Backlog 승격

사용자가 backlog 항목의 구현을 명시적으로 요청하면 다음을 함께 처리한다.

1. 관련 문서를 `docs/not-implemented/`에서 활성 문서로 옮기거나 활성 문서에 반영한다.
2. 남은 미구현 부분만 backlog에 유지한다.
3. 코드와 테스트를 활성 스펙에 맞춰 구현한다.
