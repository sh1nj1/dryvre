# 블록 편집기 스펙

## 목적과 경계

편집기는 블록의 `body_md`를 읽고 수정하는 독립 UI 모듈이다. 문서, 보드와 스트림 뷰가 같은 편집기를 재사용한다.

편집기가 블록 트리, 태스크, 메시지나 문서에 대한 별도 데이터 모델을 소유해서는 안 된다. 트리 구조 변경은 블록 연산으로 요청하고, 본문의 유일한 저장 형식은 Markdown 문자열이다.

## 불변 조건

- 정본은 항상 `body_md`다.
- 렌더링 HTML을 다시 Markdown으로 직렬화하지 않는다.
- 편집기 전용 JSON 문서 모델을 데이터베이스에 저장하지 않는다.
- 제목, 목록, 인용, 코드와 링크는 Markdown에서 파생한다.
- 태스크 상태, 부모, 순서와 참조 관계는 본문과 별도인 블록 속성이다.
- 뷰가 달라도 같은 블록 ID와 편집 계약을 사용한다.

## 표시 모드

### 읽기 모드

Markdown을 안전한 HTML로 렌더링한다. 블록 전체를 선택할 수 있고 링크, 코드와 참조 칩은 각자의 상호작용을 유지한다.

### 편집 모드

선택한 블록 하나만 자동 높이 입력면으로 바꾼다. 입력면은 원본 Markdown 문자열을 직접 편집한다. `textarea` 또는 plaintext 전용 `contenteditable` 중 어느 것을 사용해도 되지만 HTML을 정본으로 취급해서는 안 된다.

진입 방법:

- 블록 본문 클릭
- 선택된 블록에서 `Enter`
- 새 블록 생성 직후 자동 진입

종료 방법:

- `Cmd/Ctrl + Enter`: 즉시 저장 후 읽기 모드
- 포커스가 블록 밖으로 이동: 변경 내용 저장
- `Escape`: 마지막 서버 저장 상태로 되돌리고 읽기 모드

## 컴포넌트 계약

구현체는 개념적으로 다음 입력과 이벤트를 제공한다.

```ts
type BlockEditorProps = {
  blockId: string;
  bodyMd: string;
  version: number;
  readOnly: boolean;
  autoFocus?: boolean;
  onEdit(input: { id: string; bodyMd: string; version: number }): Promise<SaveResult>;
  onCreateAfter(input: { id: string; bodyMd: string }): Promise<Block>;
  onIndent(id: string): Promise<void>;
  onOutdent(id: string): Promise<void>;
  onDelete(id: string): Promise<void>;
  onReference(input: { fromId: string; toId: string }): Promise<void>;
};
```

뷰는 저장 방법을 직접 구현하지 않고 이 계약을 통해 블록 연산을 요청한다.

## 입력과 저장

1. 편집 진입 시 서버에서 받은 `body_md`와 `version`을 로컬 초깃값으로 둔다.
2. 키 입력은 로컬 문자열에 즉시 반영한다.
3. 입력이 잠시 멈추면 500~800ms debounce 후 `edit(id, body_md, version)`을 보낸다.
4. 포커스 이탈과 `Cmd/Ctrl + Enter`에서는 debounce를 기다리지 않고 저장한다.
5. 요청 중 추가 입력이 생기면 새 값을 잃지 않고 이전 요청 완료 후 최신 값만 다시 보낸다.
6. 서버가 새 `version`을 반환하면 이후 요청은 그 버전을 사용한다.
7. 저장 상태는 `Saving`, `Saved`, `Conflict`, `Offline` 정도로만 표현한다.

빈 문자열도 유효한 임시 편집 상태다. 포커스를 벗어날 때 빈 블록을 자동 삭제할 수 있지만, 네트워크 오류 중에는 삭제하지 않는다.

## 키보드 규칙

| 입력 | 동작 |
| --- | --- |
| `Enter` | 커서 위치에서 현재 본문을 나누고 다음 형제 블록 생성 |
| `Shift + Enter` | 같은 블록 안에 줄바꿈 삽입 |
| `Alt + Enter` | 현재 본문을 유지하고 바로 아래에 빈 형제 블록 생성 |
| fenced code 안의 `Enter` | 같은 블록 안에 줄바꿈 삽입 |
| `Tab` | 현재 블록을 이전 형제의 마지막 자식으로 이동 |
| `Shift + Tab` | 현재 블록을 부모 다음의 형제로 이동 |
| 빈 블록의 `Backspace` | 현재 블록 삭제 후 이전 블록 끝으로 포커스 이동 |
| 본문 시작의 `Backspace` | MVP에서는 구조를 변경하지 않음 |
| `Cmd/Ctrl + Enter` | 즉시 저장하고 편집 종료 |
| `Escape` | 로컬 변경 취소 |
| `Cmd/Ctrl + B` | 선택 범위를 `**`로 감싸기 |
| `Cmd/Ctrl + I` | 선택 범위를 `*`로 감싸기 |
| `Cmd/Ctrl + K` | Markdown 링크 입력 보조 |
| `@` | 사람, 에이전트와 블록 참조 피커 열기 |

`Enter`로 분할할 때 커서 앞 문자열은 현재 블록에 남고 뒤 문자열은 새 형제 블록의 `body_md`가 된다. 새 블록은 현재 블록 바로 다음 정본 순서를 얻으며 `status`는 기본적으로 `null`이다. 태스크를 연속 생성하는 전용 UX가 필요해질 때만 상태 상속을 별도 결정한다.

블록 사이 삽입 영역은 hover 또는 키보드 focus 시 `+`를 표시한다. 선택하면 그 위치에 빈 형제 블록을 만들고 즉시 편집 모드로 진입한다. 문서 흐름이 움직이지 않도록 평상시에도 최소 hit area는 유지한다.

한글을 비롯한 IME 조합 중에는 `Enter`, `Backspace`와 단축키로 블록 연산을 실행하지 않는다. `compositionend` 이후의 확정 입력만 구조 명령으로 처리한다.

## Markdown 지원 범위

MVP 필수 지원:

- 제목 `#`~`######`
- 굵게와 기울임
- 인라인 코드
- fenced code block
- 순서·비순서 목록
- 인용
- Markdown 링크

이미지 크기 조절, 표 전용 UI, Mermaid, 자동 언어 감지, 임베드와 리치 미디어 핸들은 편집기 코어에 포함하지 않는다.

Markdown 렌더러는 raw HTML을 기본적으로 비활성화하거나 sanitize해야 한다. 링크는 안전한 프로토콜만 허용한다.

## 참조 피커

`@` 입력 시 하나의 피커에서 사람, 에이전트와 블록을 검색한다.

- 사람·에이전트 선택은 멘션 표현을 본문에 삽입한다.
- 블록 선택은 읽을 수 있는 라벨을 본문에 삽입하고 `ref(from, to)` 연산을 함께 보낸다.
- 본문 라벨과 `ref` 관계 중 하나만 성공한 상태가 남지 않도록 실패 시 사용자에게 재시도 상태를 표시한다.
- 참조된 블록은 읽기 모드에서 칩 또는 링크로 렌더링하고 대상에는 backlink를 표시한다.

에이전트 멘션은 실행 주체를 명시하지만 멘션만으로 일반 문서 초안을 실행하지 않는다. 작업 블록이 `todo`로 전이되어야 실행 트리거가 된다. 에이전트가 사용자에게 보내는 질문과 승인 요청은 사용자의 Inbox 블록 아래에 생성하고 원래 작업을 `ref`한다. 자세한 계약은 [에이전트 작업 루프](agent-loop-spec.md)를 따른다.

본문에서 텍스트를 지웠다는 이유만으로 `ref`를 즉시 삭제하지 않는다. 명시적인 참조 제거 동작으로 `unref`를 보낸다.

## 뷰별 사용법

### 문서 뷰

본문 전체를 인라인 편집한다. 블록 분할, 들여쓰기와 내어쓰기를 모두 지원한다. 편집 중인 블록 외에는 렌더링 상태를 유지한다.

### 보드 뷰

카드 제목 또는 본문 영역에 같은 편집기를 사용한다. 열 이동은 편집기의 책임이 아니라 `setStatus` 또는 `move`를 호출하는 보드의 책임이다.

### 스트림 뷰

메시지 블록은 같은 방식으로 편집할 수 있다. 새 메시지 컴포저도 Markdown 문자열을 만들지만 `rank: null`인 자식 블록을 생성한다.

## 충돌과 오프라인

서버의 `version`이 편집 시작 버전과 다르면 자동 덮어쓰지 않는다.

충돌 UI는 다음을 제공한다.

- 서버의 최신 Markdown
- 사용자의 로컬 Markdown
- 내 변경 다시 적용
- 서버 버전 사용
- 두 내용을 복사해 수동 병합

네트워크가 끊기면 현재 입력을 메모리에 유지하고 `Offline`을 표시한다. MVP에서는 영구 로컬 큐나 CRDT를 구현하지 않는다. 페이지 이탈 전에 저장되지 않은 변경이 있으면 경고한다.

브라우저의 로컬 텍스트 undo/redo는 편집 세션 안에서 허용한다. 서버 `op_log` 기반의 블록 연산 실행 취소와는 별개다.

## 접근성

- 모든 블록은 키보드로 선택하고 편집 모드에 진입할 수 있어야 한다.
- 편집 상태와 저장 상태를 보조 기술에 전달한다.
- 포커스 이동 후 현재 블록을 예측 가능하게 유지한다.
- 색상만으로 선택, 저장 실패나 충돌을 표현하지 않는다.
- 참조 피커는 combobox/listbox 키보드 패턴을 따른다.

## MVP 완료 기준

- 문서 뷰에서 클릭과 키보드로 블록을 편집할 수 있다.
- 새로고침 후 Markdown이 손실 없이 유지된다.
- `Enter` 분할, `Shift + Enter` 줄바꿈과 빈 블록 삭제가 작동한다.
- 한글 IME 입력 중 의도하지 않은 블록 생성이나 삭제가 없다.
- 저장 중 추가 입력이 유실되지 않는다.
- 세 뷰가 같은 편집 결과를 표시한다.
- 버전 충돌이 조용히 덮어써지지 않는다.

## 비범위

- Google Docs 수준의 문자 단위 동시 편집
- CRDT 또는 Operational Transform
- 리치 HTML을 정본으로 하는 WYSIWYG 문서 모델
- 플러그인 마켓과 임의 노드 타입
- 표, 슬라이드, 캔버스와 스프레드시트 전용 편집기
- 모바일 네이티브 입력 엔진
- 계층적 권한 관리와 권한별 편집 제어
