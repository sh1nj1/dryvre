# UI 규칙

현재 인터랙티브 목업은 [`demo/mockup/index.html`](../demo/mockup/index.html)이다. 목업의 붉은 점선은 해커톤 MVP에서 구현하지 않는 완성형 아이디어를 구분하기 위한 주석이며 실제 제품 UI가 아니다.

## 전체 레이아웃

- 왼쪽: 유일한 블록 트리와 검색 진입점
- 가운데: 선택한 블록을 문서, 보드 또는 스트림으로 렌더링하는 작업면
- 오른쪽: 선택한 블록, 메시지 유무, AI 입력 범위와 참조를 설명하는 인스펙터
- 상단: 현재 트리 경로와 세 가지 뷰 전환

사용자의 Inbox는 왼쪽 트리 위에 고정된 진입점으로 표시하되 실제 대상은 해당 사용자에게 연결된 하나의 블록이다. 선택하면 가운데 작업면을 그 블록의 스트림 뷰로 연다.

## 트리와 범위

- 최상위 행이 트리의 루트이자 작업 공간이다.
- 워크스페이스 스위처를 두지 않는다. 루트 트리와 역할이 중복되기 때문이다.
- 프로젝트, 에픽, 이슈를 위한 고정 아이콘, 배지, 컨테이너나 전용 화면을 요구하지 않는다.
- 왼쪽 트리에서 블록을 선택하면 그 블록이 현재 범위와 AI 초점이 된다.
- 별도의 `Subtree` 범위 버튼을 두지 않는다. 트리 선택과 상태가 중복되기 때문이다.

## 검색과 필터

- 상단 전역 검색 버튼을 두지 않는다.
- 왼쪽 트리의 `Search & filter`가 유일한 검색 진입점이다.
- 검색 버튼을 누르면 텍스트, 참조, 상태, 작성자와 기간 조건을 조합하는 종합 팝업을 연다.
- 검색 범위는 현재 선택한 트리 경로에서 자동 결정한다.
- 결과는 별도 검색 페이지나 평평한 목록이 아니라 필터된 트리 모양을 유지한다.
- 필터된 결과에서도 블록을 바로 편집할 수 있어야 한다.
- 별도 `Filter` 및 `Sort` 툴바 버튼은 두지 않는다.

## 세 가지 뷰

세 뷰는 동일한 독립 편집기 모듈을 재사용한다. 본문 편집의 상세 동작과 저장 계약은 [블록 편집기 스펙](editor-spec.md)을 따른다.

### 문서

- 정본 블록을 재귀적으로 렌더링한다.
- 스트림 메시지를 문서 본문 중간에 `3 messages` 같은 행으로 삽입하지 않는다.
- 본문 편집은 Markdown 정본을 훼손하지 않아야 한다.

### 보드

- 현재 범위에서 `status`가 있는 블록만 표시한다.
- 열은 `To do`, `In progress`, `Blocked`, `Done` 네 개이며 모든 뷰와 검색에서 같은 네 상태를 사용한다.
- 열 이동이나 상태 변경은 같은 블록의 `status`만 바꾼다.
- 별도 카드나 이슈 레코드를 생성하지 않는다.

### 스트림

- 현재 선택한 블록에 붙은 메시지를 시간순으로 표시한다.
- 작성한 메시지는 해당 블록의 자식 블록으로 저장한다.
- 답글도 별도 스레드가 아니라 메시지 블록의 자식이다.
- 컴포저는 하나만 두고 현재 선택한 블록에 바인딩한다.
- Inbox 스트림의 에이전트 요청은 요청한 에이전트, 원래 작업 링크, 필요한 질문 또는 승인 내용을 함께 보여준다.
- 사용자는 요청에 답글을 달아 정보를 제공하거나 승인을 명시한다. 답변을 위해 별도 모달이나 승인 엔티티를 만들지 않는다.

## 에이전트 작업 루프

- 사용자가 스트림에서 `@PM Agent`에게 요청하면 PM Agent는 현재 컨텍스트를 읽고 결과물, 완료 조건, 제약, 검증 방법과 `@Developer Agent`를 포함한 작업 블록을 만든다.
- PM Agent가 만든 블록은 상태 없는 초안이다. 사용자가 내용을 검토하고 `todo`로 바꾸기 전에는 어떤 실행 에이전트도 시작하지 않는다.
- Developer Agent가 완료 계약을 충족한 작업을 선점하면 같은 블록이 `in_progress`로 바뀐다. 별도 Agent Job 카드를 만들지 않는다.
- 완료 계약이 부족하면 Inbox 질문을 만들면서 블록을 `todo`에서 `blocked`로 옮긴다. 답글을 받으면 `todo`로 복귀해 다시 검증한다.
- 실행 중 승인이 필요하면 Inbox 요청을 만들면서 블록을 `in_progress`에서 `blocked`로 옮긴다. 답글을 받으면 `in_progress`로 복귀해 같은 실행을 재개한다.
- `Blocked` 카드에는 기다리는 Inbox 요청으로 이동할 수 있는 링크를 표시한다.
- 결과와 검증 근거가 작업 아래에 기록되고 검증이 성공한 경우에만 같은 블록을 `done`으로 바꾼다.
- 상태 변화, 에이전트 작성 블록과 Inbox 요청은 WebSocket 변경으로 현재 화면에 나타나야 하며 데모에서 수동 새로고침을 요구하지 않는다.

## 오른쪽 인스펙터

- 선택된 블록의 경로, 제목과 요약을 표시한다.
- 해당 블록에 메시지가 있으면 메시지 수와 최근 참여자를 카드로 표시한다.
- 메시지 카드를 누르면 선택 상태를 유지한 채 스트림 뷰로 전환한다.
- AI가 읽을 현재 블록, 조상·자손 범위와 명시적 참조를 눈에 보이게 표시한다.
- AI 컨텍스트를 켜고 끄는 별도 고급 설정은 두지 않는다.

## 제거된 중복 UI

- 워크스페이스 스위처: 트리의 루트와 중복
- 상단 전역 검색: 트리 검색과 중복
- `Subtree`: 선택된 블록이 이미 범위를 결정
- 독립 `Filter`: 종합 검색 팝업에 통합
- `Sort`: 초기 제품의 핵심 흐름이 아니며 정본 순서를 불명확하게 함
- `Share`: 해커톤 MVP 범위 밖이며 핵심 온톨로지를 설명하지 않음
- 참여 사용자 목록과 실시간 presence: MVP 이후 협업 기능으로 검토하며 현재 목업에서는 숨김
- 실제 조작 없이 범위 표식과 더보기만 담는 보조 툴바: 화면에서 제거
- 문서 본문의 메시지 접힘 행: 문서 구조를 방해하므로 인스펙터로 이동

## 상호작용 일관성

- 뷰를 전환해도 선택된 블록은 유지한다.
- 모든 뷰에서 같은 제목, 상태, 참조와 메시지 수를 보여준다.
- 어떤 조작이 새 엔티티를 만드는지, 기존 블록의 속성을 바꾸는지 사용자가 예측할 수 있어야 한다.
- UI 문구는 블록, 트리, 참조라는 제품 언어를 우선한다.
