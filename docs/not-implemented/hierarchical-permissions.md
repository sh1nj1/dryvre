# 계층적 권한 관리 (TODO)

> 상태: 해커톤 이후 검토. 현재 구현 범위가 아니다.

해커톤 MVP는 신뢰할 수 있는 단일 사용자 데모 환경을 전제로 한다. 블록별 접근 제어와 트리 상속을 해커톤까지 안전하게 완성할 수 없으므로 스키마, API, 서버 판정, UI와 테스트를 현재 완료 기준에서 제외한다.

## 후보 설계

다음 항목은 향후 검토할 후보이며 확정된 구현 계약이 아니다.

- 최소 권한 단계를 `read`, `write`, `manage`로 제한한다.
- 부모 권한을 자식이 상속하고 필요한 하위 블록에서 명시적으로 덮어쓴다.
- UI 노출과 별개로 서버가 모든 조회와 변경의 최종 권한을 판정한다.
- 별도 권한 캐시를 도입하기 전에 부모 체인 또는 materialized path 기반 판정 비용을 측정한다.
- 블록 이동, 참조, 검색, Inbox와 Agent 실행에서 정보가 새지 않는 규칙을 함께 정의한다.

## 활성 범위로 승격할 조건

- 사용자와 Agent identity의 신뢰 경계가 정의되어 있다.
- 읽기 불가 조상 아래의 자손, backlink와 검색 결과 노출 규칙이 정해져 있다.
- 생성, 이동, 편집, 삭제와 참조 연산별 권한 행렬이 있다.
- 권한 변경 및 상속 덮어쓰기 UX가 합의되어 있다.
- 서버 통합 테스트와 권한 우회 회귀 테스트를 완료 기준에 포함할 수 있다.

이 항목을 구현할 때는 이 문서를 활성 스펙으로 옮기거나 관련 활성 문서에 반영한 뒤 코드와 마이그레이션을 변경한다.
