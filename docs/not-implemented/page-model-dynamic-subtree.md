# 대안 A: 동적 서브트리 페이지

## 요약

별도의 페이지 타입을 만들지 않는다. 사용자가 어떤 블록을 선택하든 그 블록이 현재 문서의 루트가 되고, 정본 자손 블록이 페이지 콘텐츠가 된다.

페이지는 저장된 엔티티가 아니라 선택된 블록 서브트리의 문서 투영이다.

## 구조

```text
Product Studio
├─ Build Week Launch
│  ├─ Product thesis
│  ├─ Core experience
│  │  ├─ Three views, one tree
│  │  └─ AI writes back
│  └─ Launch checklist
└─ User research
```

`Build Week Launch`를 선택하면 그 블록이 제목이 되고 아래의 모든 정본 자손이 문서 본문이 된다. `Core experience`를 선택하면 같은 데이터 중 해당 서브트리만 새 문서 범위로 렌더링한다.

## 핵심 규칙

- 모든 블록은 주소를 가질 수 있고 문서 루트가 될 수 있다.
- 선택된 블록이 문서, 보드, 스트림과 AI 컨텍스트의 현재 범위를 결정한다.
- 문서 뷰는 현재 블록 아래에서 `rank`가 있는 블록만 재귀적으로 렌더링한다.
- `rank`가 `null`인 스트림 메시지는 문서 본문에 포함하지 않는다.
- 블록을 페이지로 변환하거나 페이지에서 해제하는 별도 연산은 없다.
- 데이터 이동 없이 선택만 바꾸어 더 좁은 서브트리를 독립된 문서처럼 열 수 있다.

## 큰 서브트리의 표시

루트나 큰 블록을 선택하면 한 화면에 너무 많은 블록이 표시될 수 있다. 다음과 같은 점진적 공개가 필요하다.

- 루트에서는 기본적으로 직계 자식 중심의 개요를 보여준다.
- 일반 블록은 정해진 상대 깊이 또는 렌더링 예산까지만 먼저 보여줄 수 있다.
- 생략된 자손이 있는 경계에는 `N개 하위 블록 더 보기` 같은 이어보기 행을 표시한다.
- 이어보기 행을 누르면 현재 문서 안에서 추가 자손을 불러온다.
- 블록 제목을 누르면 해당 블록을 새 문서 루트로 연다.
- 펼침 상태는 화면 상태이며 블록 데이터에는 저장하지 않는다.

모든 블록에 펼침 아이콘을 상시 표시하지 않고, 실제로 자손을 생략한 지점에만 이어보기 표시를 둔다.

## 왼쪽 트리

- 하나의 정본 블록 트리를 탐색한다.
- 루트부터 선택 블록까지의 조상 경로를 유지한다.
- 선택 블록의 직계 자식까지 기본으로 펼치고 더 깊은 가지는 사용자가 요청할 때 펼친다.
- 문단과 같은 작은 블록까지 모두 표시하면 탐색 밀도가 높아질 수 있으므로 검색, 접기와 가상화가 중요하다.

## 데이터 모델과 연산

기존 블록 모델을 그대로 사용한다.

```text
block(id, parent_id, path, rank?, body_md, status?, ...)
```

페이지 여부를 저장하지 않으므로 기존의 `create`, `move`, `edit`, `setStatus`, `ref`, `unref`, `delete` 연산만 필요하다.

## 장점

- 블록, 트리와 참조라는 기존 제품 온톨로지를 그대로 유지한다.
- 어떤 블록도 즉시 공유 가능한 문서와 AI 컨텍스트가 될 수 있다.
- 페이지 전환이나 폴더 이동에 따른 구조 제약이 없다.
- 문서, 보드와 스트림이 동일한 선택 범위를 공유한다.
- 깊은 링크와 부분 공유가 자연스럽다.

## 위험과 비용

- 루트나 큰 서브트리에서 문서가 지나치게 길어질 수 있다.
- 자동 펼침 깊이나 렌더링 예산이 사용자에게 임의적으로 느껴질 수 있다.
- 숨겨진 자손의 존재와 펼치는 방법을 문서 안에서 설명해야 한다.
- 왼쪽 트리가 문서의 모든 콘텐츠 블록까지 담으면 전통적인 파일 탐색기보다 복잡하다.
- 페이지라는 익숙한 사용자 개념과 실제 선택 범위가 일치하지 않을 수 있다.

## 검증할 질문

- 기본 렌더링 깊이와 블록 수 예산을 어떻게 정할 것인가?
- 루트 개요가 일반 문서 뷰와 얼마나 달라도 같은 뷰로 이해되는가?
- 사용자는 블록 인라인 펼치기와 블록을 새 루트로 여는 동작을 구분할 수 있는가?
- 검색 결과로 깊은 블록을 열었을 때 주변 문맥을 어느 범위까지 보여줄 것인가?
